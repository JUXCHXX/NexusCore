## ВҝQuГ© es NexusCore?

NexusCore es una plataforma SaaS **production-ready** de gestiГіn de proyectos con IA integrada, construida para equipos de desarrollo modernos. Combina tableros Kanban con drag & drop, vistas Gantt con dependencias, y un asistente IA que analiza tu sprint, detecta bloqueos y sugiere prioridades вҖ” todo en un solo workspace oscuro y veloz.

> Construido como proyecto de portafolio fullstack demostrando arquitectura multi-tenant, integraciГіn de IA, sistema de pagos, API pГәblica, CI/CD y deploy en producciГіn.

### Multi-tenancy

Cada workspace tiene su propio **subdomain** (`miequipo.nexuscore.app`). El Edge Middleware de Vercel detecta el subdomain, extrae el slug, reescribe la URL internamente y adjunta el contexto de workspace a cada request вҖ” sin que el usuario note nada.

```
miequipo.nexuscore.app/board
    вҶ’ Edge Middleware detecta subdomain "miequipo"
    вҶ’ Reescribe a /app/miequipo/board
    вҶ’ Adjunta x-workspace-slug: miequipo al header
    вҶ’ tRPC context verifica membresГӯa del userId al workspace
```

## рҹҡҖ Setup local

### Prerequisitos

- Node.js 20+
- npm 10+
- Cuenta en: Neon В· Clerk В· Groq В· Pusher В· Upstash В· Resend В· Stripe

### 1. Clonar el repo

```bash
git clone https://github.com/tuusuario/nexuscore.git
cd nexuscore
```

### 2. Instalar dependencias

```bash
npm install
```

### 3. Configurar variables de entorno

```bash
cp .env.example .env.local
```

## рҹӨ– IntegraciГіn con IA

NexusCore usa **Groq** con el modelo `llama-3.3-70b-versatile` para tres funcionalidades principales:

### 1. Resumen de sprint
Analiza todas las tareas del sprint y genera un informe ejecutivo en markdown con progreso, completadas, en progreso y bloqueos detectados.

### 2. Sugerencia de prioridades
Examina el conjunto de tareas activas considerando fechas de vencimiento, carga del equipo y dependencias, y propone reordenamiento de prioridades con justificaciГіn.

### 3. DetecciГіn de bloqueos
Identifica patrones problemГЎticos: tareas sin assignee cerca del deadline, dependencias circulares, tareas bloqueadas sin acciГіn, y entrega sugerencias concretas de resoluciГіn.
